## [CH03_08] (인프라) AWS Fargate를 통한 배포
- ECS
- 클러스터
  - 컨테이너를 묶어주는 영역

### 작업 절차
- 클러스터 생성
- 작업 생성
- 작업 실행

### 클러스터 생성
- 네트워킹 전용(AWS Fargate)
- 클러스터 이름 지정
- 컨테이너 인사이트 활성화
- 생성 완료

### 클러스터 선택 - 작업 추가
- FARGATE
- 작업 정의 이름 추가
- 작업 메모리
- 작업 CPU
- 컨테이너 추가
  - 컨테이너명 추가(django)
  - 이미지: 기존 만들었던 이미지 추가
  - 메모리 제한: 128MB
  - 포트: 8000
  - 완료
- 생성

### 작업 실행
- FARGATE
- 클러스터 선택
- 작업 개수 1
- VPC 기본 지정
- 보안 그룹
  - 인바운드 규칙 8000 추가
- 작업 실행
- PENDING // 작업 실행 준비
- 작업이 완료되면 생성된 EC2 없이 정상 서비스 구동
  - EC2가 필요 없어짐
